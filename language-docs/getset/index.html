    Property Keywords

    get (accessor)

    The get keyword defines an accessor method in a property. This means that the method will be called every time the property's value is retrieved. The get accessor must return a value, and the value must be of the same type as the property.

    set (accessor)

    The set keyword defines an accessor method in a property. This means that the method will be called every time the property's value is assigned. The get accessor takes one input parameter, value, which is the assigned value.

    public class MyClass
{
    private int property = 0;

    public int MyProperty
    {
        get
        {
            mc.WriteLine("Property get called!");
            return property - 10;
        }
        set
        {
            property = value + 10;
            mc.WriteLine("Property set called: " + property);
        }
    }
}

public static class Main
{
    static void main()
    {
        MyClass a = new MyClass();
        a.MyProperty = 10;
        mc.WriteLine(a.MyProperty);
    }
}

    This will output:

    1
2
3
    Property set called: 20
Property get called!
10

    Explanation

    When the value MyProperty was assigned, the set accessor was called. In this example, it took the value, and added 10 to it. It then stored the result in a private field named property of type int. When the value of MyProperty was retrieved via mc.WriteLine(), the get accessor was called. The accessor returns the value within the private field, subtracted by 10. Since 10 was added when the value was stored, and 10 was subtracted when the value was accessed, the initial value remains untouched.

    Bad Code Practises

    It is not highly recommended that the object's state not be tampered with during the get method. For example:

    int PrivateField = 0

public int MyProperty
{
    get
    {
        return PrivateField++;
    }
}

    Every time the get accessor method is called, the object's state is changed as PrivateField increments by 1 each time.
