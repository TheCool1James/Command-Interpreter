The ILanguage

THE BASICS:


Firstly, you need to familiarize yourself with objects. An abstract object is like a real object without the specific details. Let's take a Minecraft player for instance. An abstract player would have the attributes like Name, UUID, Health, Skin, Hunger, XP, PermissionLevel. Notice how the attributes doesn't have any values assigned to it. Now, when a real player is created (an instance), values would be assigned to each attribute, like Name -> "TheCool1James", UUID -> 19DSEWFDF...., etc. Programmers use abstract objects to create real objects. Wrapped your head around that? Well, let's move on to inheritance. So, let's say you have an abstract entity, with attributes like Health, Hunger etc. Now, we create a player, inheriting the abstract entity. So, we give the players specific attributes like Skin, Username, etc. Since the player inherits the entity object, it will also have all the attributes of an entity.
Still alive? Great! Let's get coding.
So, we first need to define a target entity/player by creating a selector object. Syntax:
[object] [objectName];
 So to create an all players (@a for u minecraft ppl) selector:
aPlayer player = new aPlayer(1, 1, 1, 1);
Great! Now, we need to do something to the player.... let's kill it! So:
player.kill();
We have officially murdered an innocent player at coordinates (1, 1, 1)! Be proud.
Oh, and for all you curious people, this is how the interpreted minecraft command looks:
/kill @a[x=1, y=1, z=1, r=1]
Note how even if we didn't specify a player.r value, the interpreter defaults to the value 1. If you want a different selector, say @p, just change the object from aPlayer to pPlayer. For @r, its rPlayer. And for @e its aEntity.
To comment, simple put // at the start of the comment. The comment will last until the EOL marker, kinda like assembly code now that I think about it :)
This is code. //This is comment. The interpreter will ignore anything here.
To destroy an object, simply put:
[object].destroy();
PLEASE NOTE that no selectors can be destroyed i.e., aPlayer, rPlayer, pPlayer and aEntity. This is because there can only be one instance of every selector in the script. Previous selectors of the same type will be destroyed automatically when a new one is created. However, with any other object, i.e., Item, Entity etc., you may have multiple instances of them (with different names of course), thus you can destroy them.
One last thing: Make sure each statement ends with a semicolon, like in C#, Java or C;

DECLARE VARIABLES. NOT WAR.


To declare a variable is simple.
[type] [name] = [value];
Where [type] can be: int or string, [name] can be whatever your heart desires, and [value] must be of [type]. Example:
int fooBar = 1;
Worth Noting: You cannot put a [player/entity object]'s x, y or z (or any attributes*) for the values of the variables, UNLESS you have a [LOCAL] tag before. More on that later :(

Instances and Objects


To create a new object, you simply follow the syntax:
[type] [name] = new [type](arg[0], arg[1], ....);
 Simple.

WHILE AND IF STATEMENTS


The title probably speaks for itself. This section is dedicated to the WHILE IF statements. Let's start with the if statement. Usage:
if(condition){
code here
}
Where condition is the condition that must be true for the stuff inside of the IF to be executed. Simple right? Yeah.
Now, WHILE statements is just a repeating IF statement. So:
while(condition){
code here
}
While the condition is true, the code inside of the curly brackets will be executed. If its false, then the loop will be terminated.
Worth Noting: For IF statements, ELSE IF and ELSEs are not supported yet. (awwwwww).

Commands!!


In order to execute a command, we need a target to execute it on. So, we need to create a target object:
aPlayer p = new Player(1, 1, 1, 1);
 And then we need to specify the command we want to perform on the said object:
p.kill();
 However, if you want to execute a minecraft command that takes arguments/input, you need to specify the arguments within the bracks, like any C method.
p.say("hi", "white");
This will say "hi" to the player in white text. With more complicated commands like the give command, we need to create an object for the item we are giving:
Item i = new Item("stone", 1, 1, "name");
p.give(i);
i.destroy(); //Speeds up compiling, and reduces risk of crashes.
 We just gave the player 1 block of stone:1 named "name".
For more complex commands such as the /tellraw where there's a loooottttt of options, we have something called "overloads." This means that some optional arguments may be left blank with the null code.
[target].tellraw(text[]("text", extra (new text[]), "color", bold (bool), underline (bool), italics (bool), strikethrough (bool), obfuscated (bool), "insertation", new clickEvent(action, "value"), new hoverEvent(action, "value", translate, with), new score("name", "objective", value), "selector"));
Now you may thank Minecraft for being this complicated. The text[] thing means a text object array (not to be confused with a string array). A text object is just the universal format for all text-displaying commands in Minecraft. The array part allows you to have as many text passages as the computer would allow.
To execute a command that does not exist inside the IDE (or for some other reason), you can use the _mc() function.
_mc("/give @a stone 1 0");
Please note that the commands will be inserted AS-IS, and NO INTERPRETATION will be applied. Can cause bugs and errors when not used appropriately.

IFs and Booleans


When you type:
int i = 3; if(i == 4)
{
_mc("say @p \"hi\"");
}
 We know that the "hi" will never be said because i can never equal to 4. So, i == 4 is false. This makes the statement i==4 into a boolean (it outputs a result that is a boolean). So... what? Well, ALL Minecraft commands outputs a result that is a boolean, but its not what you think. When you place a comparator behind a command block, you can see the output for yourself. If the comparator lights up, the output is true, and if it doesn't, its false. Applying this logic, you TECHNICALLY can use IF statements on them. But again, its not what you think. Let's take target.playSound() as an example:
aPlayer p = new aPlayer(1, 1, 1, 1);
if(p.playSound("someSound", [some other arguments so here]))
{
p.kill();
}
 You would expect the player would die when the sound is played, and you would be correct. HOWEVER, when you do IF statements in this fasion, the statement WILL BE EXECUTED. So, the sound "somesound" WILL PLAY! This is the same with variable declarations. If you set a bool to the value of the command, whenever you use the boolean, THE COMMAND WILL BE EXECUTED (although this feature is not avaiable yet, and will most likely not be).

IL2JSON (IL2ALI)


CBIL is actually an IL2JSON parser thingy, which means Interpreted Langauge 2 JSON (the language used by minecraft command blocks), however, that makes no sense because JSON is ALSO an interpreted language... So... its IL2AIL (Interpreted Langauge 2 Another Interpreted Langauge). Anyways... that aside, since this is an interpreted langauge, the syntax is a little more loose than normal languages. So, don't be alarmed if an error in your code doesn't doesn't show up as an actual error. It's probably the parser's fault. If that happens, you can post a bug thing on GitHub (with your code, and an explanation about the bug) and I'll try my best to fix it.
