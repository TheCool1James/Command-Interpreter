The ILanguage

THE BASICS:


Firstly, you need to familiarize yourself with objects. An abstract object is like a real object without the specific details. Let's take a Minecraft player for instance. An abstract player would have the attributes like Name, UUID, Health, Skin, Hunger, XP, PermissionLevel. Notice how the attributes doesn't have any values assigned to it. Now, when a real player is created (an instance), values would be assigned to each attribute, like Name -> "TheCool1James", UUID -> 19DSEWFDF...., etc. Programmers use abstract objects to create real objects. Wrapped your head around that? Well, let's move on to inheritance. So, let's say you have a abstract entity, with attributes like Health, Hunger etc. Now, we create a player, inheriting the abstract entity. So, we give the players specific attributes like Skin, Username, etc. Since the player inherits the entity object, it will also have all the attributes of an entity.
Still alive? Great! Let's get coding.
So, we first need to define a target entity/player by creating a player object.
aPlayer player;
Now, we need to assign values to the player:
player.x = 1;
player.y = 1;
player.z = 1;
 Great! Now, we need to do something to the player.... let's kill it! So:
player.kill();
We have officially murdered an innocent player at coordinates (1, 1, 1)! Be proud.
Oh, and for all you curious people, this is how the interpreted minecraft command looks:
/kill @a[x=1, y=1, z=1]
If you want a different selector, say @p, just change the object from aPlayer to pPlayer. For @r, its rPlayer. And for @e its aEntity.
To comment, simple put // at the start of the comment. The comment will last until the EOL marker, kinda like assembly code now that I think about it :)
This is code. //This is comment. The interpreter will ignore anything here.
One last thing: Make sure each statement ends with a semicolon, like in C# or Java or C;

DECLARE VARIABLES, NOT WAR.


To declare a variable is simple.
[type] [name] = [value];
Where [type] can be: int or string, [name] can be whatever your heart desires, and [value] must be of [type]. Example:
int fooBar = 1;
Worth Noting: You canot put [player/entity object].x, y or z for the values of the integer variables :(

WHILE AND IF STATEMENTS


The title probably speaks for itself. This section is dedicated to the WHILE IF statements. Let's start with the if statement. Usage:
if(condition){
code here
}
Where condition is the condition that must be true for the stuff inside of the IF to be executed. Simple right? Yeah.
Now, WHILE statements is just a repeating IF statements. So:
while(condition){
code here
}
So, while the condition is true, the code inside of the curly brackets will be executed. If its false, then the loop will be terminated.
Worth Noting: For IF statements, ELSE IF and ELSEs are not supported yet. (awwwwww).
